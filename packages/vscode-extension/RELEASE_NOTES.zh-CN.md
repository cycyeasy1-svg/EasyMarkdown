## v1.4.0

EasyMarkdown VSCode 扩展 v1.4.0 更新日志

本次更新重点提升了 Keep 模式的编辑安全性、表格与块操作效率、VSCode 导航体验，并新增附件插入和更多内容缩放能力。

### 新增与优化

- 增强 Keep 编辑草稿保护：未确认的单元格、块和插入草稿可在标签页隐藏或 Webview 恢复后继续编辑；保存或切换到源码模式前会确认待提交内容，外部修改导致定位冲突时也可保留并复制草稿。
- 提升表格与块编辑效率：支持键盘选择和移动表格单元格、从电子表格粘贴矩形 TSV 数据、在筛选值旁显示出现次数，并可从 Keep 右键菜单插入、复制或删除 Markdown 块。
- 新增“添加附件”命令：可将所选文件复制到文档旁的 `assets/` 目录，并在 Keep 或源码模式的当前位置插入相对 Markdown 链接。
- 加强 VSCode 集成：Keep 模式会显示 Markdown 诊断数量并可打开原生问题面板；按住 Alt/Option 点击相对链接可在侧栏打开；从搜索结果、问题面板或 file:line 导航时会保留源码模式和精确选择位置。
- 扩展内容查看能力：KaTeX 公式可使用与 Mermaid 相同的缩放、拖动和平移灯箱；安全的空 `<a id>` / `<a name>` 锚点不再显示为源码文字，并可跳转到折叠区域内的位置。
- 收紧外部链接处理：来自 Markdown 的链接仅允许通过 HTTP、HTTPS 和 mailto 协议调用系统应用。

### 问题修复

- 修复打开查找栏后继续编辑会留下过期高亮和匹配数量的问题；查找结果会实时刷新，同时保持当前结果和滚动位置。

## v1.3.4

EasyMarkdown VSCode 扩展 v1.3.4 更新日志

本次更新进一步完善了 Keep 模式的 Markdown 显示、空行排版、搜索定位和表格操作，并修复了 VSCode 差异对比与编辑模式切换中的问题。

### 新增与优化

- 更完整地显示内联 Markdown：支持 GFM 删除线、下划线强调、自动链接、链接标题、转义标点、URL 嵌套括号和 `==高亮==`，显示效果与 EasyMarkdown 富文本编辑器保持一致。
- 新增连续空行保留选项：可在编辑器设置中保留有意输入的多行空白，同时不修改 Markdown 源文件。
- 增强表格列操作：根据表头和单元格内容分配初始宽度，并支持拖动调整列宽、单列或整表自动适配，以及临时隐藏和恢复列；这些操作都不会修改 Markdown 源文件。
- 优化 VSCode 搜索结果定位：从搜索结果打开 Markdown 后，Keep 模式会定位并高亮对应内容；若匹配的是未直接渲染的 Markdown 语法，则会高亮最接近的源码块。

### 问题修复

- 修复 Keep 模式接管 VSCode SCM 差异编辑器的问题；差异对比继续使用源码视图，也不会显示模式切换操作。
- 修复记住编辑模式后切换可能留下重复标签页的问题；预览标签页会原位复用，多余的固定标签页会被正确关闭。
