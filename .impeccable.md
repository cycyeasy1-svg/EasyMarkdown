## Design Context

### Users
EasyMarkdown は、Markdown 原文の差分を崩さずに文書を閲覧・編集したいライター、開発者、および Git ベースで共同作業する利用者を対象とする。デスクトップ App と VS Code 拡張のどちらでも、長い技術文書や表を素早く読み取り、必要な箇所だけを安全に編集できることを重視する。

### Brand Personality
温かい、現代的、誠実。利用者に落ち着きと信頼感を与え、原文を尊重する編集体験を提供する。

### Aesthetic Direction
Typora のように本文へ集中できる静かな編集面を基調とし、既存の暖色系テーマと Morandi 系テーマ、ライト／ダーク両方に自然に馴染ませる。常時表示の重いツールバーや過度な装飾は避け、高度な操作は対象へ近づいたときに段階的に提示する。

### Design Principles
- Markdown 原文を正本として扱い、プレビュー操作だけで不要な差分を発生させない。
- 通常の閲覧時は本文を主役にし、追加操作は必要な場所で明確かつ控えめに提示する。
- App と VS Code 拡張で同じ概念・用語・操作感を維持する。
- 長文・幅広表・多列表でも、読みやすさと応答性を損なわない。
- ライト／ダークテーマ、キーボード操作、フォーカス表示を含めて利用可能性を保つ。
