# App 用户体验改善路线图

最后更新日期：2026-07-16

## 1. 本文档的定位

本文档用于跟踪以 EasyMarkdown App 为主的操作体验与功能改善。
它整合了此前讨论中提出的方案，以及第一期完成后基于最新代码重新检查所得的追加建议；
今后的用户体验改善工作以本文档为准。

- 如果讨论中的旧优先级与本文档冲突，以本文档为准。
- 已完成项目不删除，而是继续补充完成日期、提交和验证结果。
- 新建议不强行插入既有阶段，先作为“候补”重新评估优先级。
- 以 App 为第一目标；移动端不删除功能，而是根据屏幕尺寸调整操作方式。
- 参考 VS Code 的体验，但优先保证用户对正文的专注，以及 Keep 的零差分承诺。

## 2. 用户体验改善的基本原则

1. **安全编辑**
   - 将 Markdown 原文作为唯一真实来源，不产生不必要的重新格式化或差异。
   - 不因未确认编辑、外部变更、模式切换或标签切换而丢失输入内容。

2. **以正文为主角**
   - 不过度增加常驻操作，只在需要的位置逐步显示。
   - 低频操作统一收纳到上下文菜单或命令面板。

3. **兼顾易发现性与效率**
   - 为新用户提供明确的标签、工具提示和状态显示。
   - 为熟练用户提供键盘、鼠标侧键和命令面板操作。

4. **统一操作语义**
   - 对于同类编辑，无论是单元格、块、任务还是表格操作，
     都使用一致的 Undo/Redo、草稿保护和成功通知规则。

5. **不让性能倒退**
   - 以长文档、大型表格和大量标签为前提进行设计。
   - 避免重建整个 DOM 或复制整个文件，使用局部更新与行级补丁。
   - 可能增加长任务的变更必须配套可重复的性能 benchmark。

6. **明确实现完成条件**
   - 将中、日、英 i18n、键盘操作、无障碍、单元测试和 E2E
     纳入每项功能的完成条件。

### UX-10：Keep 修改审阅与部分恢复

优先级：**P1**

状态：`DONE`

目标：

- 让用户能在保存前确认“修改了什么”。
- 让 Undo 不仅按时间顺序撤销，还能识别修改位置并恢复。

实现内容：

1. 为 Keep 历史条目添加语义信息。
   - 类型：单元格编辑、块编辑、任务切换、添加行、删除行、添加列、删除列
   - 对象：行号、表格编号、行列编号、块类型
   - 显示摘要
   - 创建时间

2. 让操作后的通知更具体。
   - 例如：“已修改单元格 · 撤销”
   - 例如：“已删除 1 行 · 撤销”
   - 通知中的 Undo 使用同一套历史 API。

3. 增加“查看本次修改”。
   - 按修改块列出当前内容与 `savedContent` 的差异。
   - 点击修改位置可跳转到正文。
   - 每个项目都可单独恢复到保存时状态。

   - 不只提供 “Keep mine / Reload”，还要能在决定前查看差异。

## 8. 第三期：同一文档的源码 ↔ Keep 同步分栏

### UX-02：“在侧边打开源码”

优先级：**P1**

状态：`DONE`

目标：

- 同时查看 Markdown 原文与渲染结果，安全编辑复杂 Markdown。
- 减少在当前“切换到源码显示”之间反复往返。

实现内容：

1. 以两种视图打开同一标签。
   - 左侧源码 / 右侧 Keep
   - 左侧 Keep / 右侧源码
   - 保留现有“在右侧打开其他标签”的分栏功能。

2. 同步
   - 将源码编辑同步到 Keep。
   - 将 Keep 已确认编辑同步到源码。
   - 使用同一份 `tab.content` 作为唯一真实来源，不持有两套文档状态。
   - 对输入期间的预览更新进行防抖，减少长文档重绘。

3. 位置同步
   - 双向滚动同步
   - 淡色标记源码光标位置对应的 Keep 块
   - 双击 Keep 或选择“在源码中打开”时聚焦对应行
   - 表格使用单元格的 `data-line` 映射行位置

- 优先保护隐私，默认关闭，只有用户在设置中明确启用后才生效。
- 保存成功时为上一次保存内容创建快照；每个文档最多 30 个版本、保留 30 天，
  全局上限为 64 MB。单个快照超过 2 MB 时不保存。
- 自动保存会合并十分钟内连续产生的快照，避免历史快速增长。
- 可从命令面板打开历史列表，执行比较、作为 Keep 事务恢复、删除单条记录或按文档删除；
  也可从设置删除全部历史。
- 历史只保存在 Electron `userData/local-history` 中，不会自动发送到原文档文件夹或云端。
  移动端设备备份策略不同，因此暂不支持。
- `test/local-history.test.js` 和 `test/e2e/local-history.spec.js` 覆盖保留限制、
  自动保存合并，以及重启后的列表、比较和恢复。
